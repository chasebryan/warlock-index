Source Registers provide the vetted, classified lists of source families used
across the corpus. They record publisher, access patterns, source class, known
limitations, and recommended use for each family or lane.

Registers are living references. They are updated as new source families are
incorporated or as access/caveat information changes. Each register entry should
preserve clear use constraints so products do not misuse sources.

When adding entries, also update linked trackers, packets, and the coverage map
as appropriate.

After register edits, re-run build+check to refresh navigation and corpus exports.

## Usage

- Consult the relevant register before using a source in a new product.
- Record source class (A/B/C/D) and any special handling.
- When a source family receives new treatment notes, link the corresponding
  source note from collections/source-notes.
- Update the register and the [Documentation Index](../index.md) together.
- Prefer official (Class A) sources as the anchor for major judgments.
